# Mengubah Product Screen

---

Klik pada `product_screen.dart`, lalu sesuaikan source code-nya menjadi seperti ini:

```dart
import 'package:flutter/material.dart';

import '../widgets/product_grid_widget.dart';

class ProductScreen extends StatelessWidget {
  const ProductScreen({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        centerTitle: true,
        title: const Text("Shop App"),
      ),
      body: const ProductGridWidget(),
    );
  }
}
```

Di sini, kita akan memisahkan perulangan yang terjadi pada `GridView`. Perulangan tersebut kita simpan di sebuah widget dengan nama `ProductGridWidget`. Ikuti langkah selanjutnya untuk membuat widget tersebut.

# Membuat Product Grid Widget

---

Pada folder `widgets`, buat sebuah file baru dengan nama `product_grid_widget.dart`. Sesuaikan source code-nya menjadi seperti ini:

```dart
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

import '../models/product_model.dart';
import '../widgets/product_widget.dart';
import '../providers/product_provider.dart';

class ProductGridWidget extends StatelessWidget {
  const ProductGridWidget({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    int crossAxisCount = 2;
    double width = MediaQuery.of(context).size.width;

    if (width < 300) {
      crossAxisCount = 1;
    } else if (width < 576) {
      crossAxisCount = 2;
    } else if (width >= 576 && width < 768) {
      crossAxisCount = 3;
    } else if (width >= 768 && width < 992) {
      crossAxisCount = 4;
    } else {
      crossAxisCount = 6;
    }

    final products = Provider.of<ProductProvider>(context).getProducts();

    return Padding(
      padding: const EdgeInsets.all(20.0),
      child: GridView.builder(
          gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
            crossAxisCount: crossAxisCount,
            mainAxisSpacing: 20.0,
            crossAxisSpacing: 20.0,
            childAspectRatio: 3 / 2,
          ),
          itemCount: products.length,
          itemBuilder: (BuildContext context, int index) {
            ProductModel product = products[index];

            return ProductWidget(product: product);
          }),
    );
  }
}
```

Bisa dilihat, kita juga memisahkan product di sebuah widget tersendiri. Sehingga, `product_grid_widget.dart` hanya berfokus pada perulangan saja, sedangkan product-nya di-handle pada sebuah file tersendiri.

Mengenai data product yang digunakan, kita dapatkan dari provider. Tentu saja provider belum dibuat, silakan ikuti langkah-langkah selanjutnya.

# Mengubah Product Widget

---

Di sini, kita akan mengubah file `product_widget.dart` agar hanya fokus menampilkan sebuah product saja. Sesuaikan source code-nya menjadi seperti di bawah ini:

```dart
import 'package:flutter/material.dart';

import '../models/product_model.dart';
import '../screens/product_detail_screen.dart';

class ProductWidget extends StatelessWidget {
  final ProductModel product;

  const ProductWidget({
    Key? key,
    required this.product,
  }) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return ClipRRect(
      borderRadius: BorderRadius.circular(10.0),
      child: GridTile(
        footer: GridTileBar(
          backgroundColor: Colors.black54,
          leading: IconButton(
            onPressed: () {},
            icon: Icon(
              Icons.favorite,
              color: Colors.blue.shade300,
            ),
          ),
          title: Text(product.name),
          trailing: IconButton(
            onPressed: () {},
            icon: Icon(
              Icons.shopping_cart,
              color: Colors.blue.shade300,
            ),
          ),
        ),
        child: GestureDetector(
          child: Image.network(product.imageUrl),
          onTap: () {
            Navigator.pushNamed(
              context,
              ProductDetailScreen.routeName,
              arguments: product.id,
            );
          },
        ),
      ),
    );
  }
}
```

Bisa dilihat, sekarang program kita menjadi lebih clean. Tentu saja akan ada error karena kita belum menambahkan provider atau routes. Silakan ikuti langkah selanjutnya.

Di sini, kita juga menambahkan `GestureDetector`. Tujuannya adalah agar product bisa diklik. Perhatikan bagian `onTap`. Ketika kita menekan product, maka kita akan berpindah pada halaman product details.

# Membuat Product Provider

---

Pada folder `lib`, buat sebuah file baru dengan nama `product_provider.dart`. Sesuaikan source code-nya menjadi seperti ini:

```dart
import 'package:flutter/material.dart';

import '../models/product_model.dart';

class ProductProvider with ChangeNotifier {
  List<ProductModel> _products = [
    ProductModel(
      id: "id-1",
      name: "Macbook Air",
      description: "Macbook Air",

      imageUrl:
          "https://images.pexels.com/photos/5202951/pexels-photo-5202951.jpeg?auto=compress&cs=tinysrgb&w=1260&h=750&dpr=1",
    ),
    ProductModel(
      id: "id-2",
      name: "Kamera Hitam dan Abu-Abu",
      description: "Kamera Hitam dan Abu-Abu",
      price: 7000000,
      imageUrl:
          "https://images.pexels.com/photos/1203803/pexels-photo-1203803.jpeg?auto=compress&cs=tinysrgb&w=1260&h=750&dpr=1",
    ),
    ProductModel(
      id: "id-3",
      name: "Flashdisk",
      description: "Flashdisk",
      price: 100000,
      imageUrl:
          "https://images.pexels.com/photos/10336136/pexels-photo-10336136.jpeg?auto=compress&cs=tinysrgb&w=1260&h=750&dpr=1",
    ),
    ProductModel(
      id: "id-4",
      name: "Keyboard",
      description: "Keyboard",
      price: 250000,
      imageUrl:
          "https://images.pexels.com/photos/585752/pexels-photo-585752.jpeg?auto=compress&cs=tinysrgb&w=1260&h=750&dpr=1",
    ),
  ];

  List<ProductModel> getProducts() {
    return _products;
  }

  void addProduct(ProductModel product) {
    _products.add(product);
    notifyListeners();
  }

  ProductModel findById(String id) {
    return _products.firstWhere((product) => product.id == id);
  }
}
```

Bisa dilihat, data product sudah kita pindahkan ke provider yang baru saja kita buat. Ketika membuat sebuah provider, jangan lupa menambahkan `with ChangeNotifier`.

Kemudian, kita buat juga bermacam-macam fungsi yang kita perlukan. Di sini, kita membuat `getProducts()`, `addProduct()` dan `findById()`. Meski begitu, pada tutorial kali ini, kita tidak akan menambahkan product apapun.

Jika kita ingin mengubah data, maka jangan lupa menambahkan `notifyListeners()`. Tujuannya adalah agar flutter melakukan render ulang, sehingga layar menjadi berubah. Lebih lengkap, silakan baca pada [dokumentasi provider](https://pub.dev/packages/provider){:target="\_blank"}.

Tentu saja ini belum selesai, kita harus menambahkan provider tersebut pada `main.dart`. Klik pada file `main.dart`, kemudian sesuaikan source code-nya menjadi seperti ini:

```dart
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

import '../screens/product_screen.dart';
import '../providers/product_provider.dart';
import '../screens/product_detail_screen.dart';

void main() {
  runApp(
    MultiProvider(
      providers: [
        ChangeNotifierProvider(
          create: ((context) => ProductProvider()),
        )
      ],
      child: const MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  const MyApp({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      title: "Shop App",
      home: const ProductScreen(),
      routes: {
        ProductDetailScreen.routeName: (context) => const ProductDetailScreen()
      },
    );
  }
}
```

Di sini, kita menggunakan `MultiProvider`, tujuannya adalah agar lebih mudah menambahkan provider baru di kemudian hari. Setiap provider yang akan digunakan harus ditambahkan pada halaman ini.

# Membuat Routes

---

Bisa dilihat pada file `main.dart` di atas, kita sudah menambahkan routes. Tujuannya adalah agar ketika sebuah product diklik, maka akan pindah ke halaman `ProductDetailScreen`. Nama dari route tersebut, kita buat pada halaman `ProductDetailScreen`.

Tentu saja file tersebut belum ada. Maka, pada folder `screens`, buat sebuah file baru dengan nama `product_detail_screen.dart`, lalu sesuaikan source code-nya menjadi seperti ini:

```dart
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

import '../providers/product_provider.dart';

class ProductDetailScreen extends StatelessWidget {
  static String routeName = "/product-detail";

  const ProductDetailScreen({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    final productId = ModalRoute.of(context)!.settings.arguments as String;

    final currentProduct =
        Provider.of<ProductProvider>(context).findById(productId);

    return Scaffold(
      appBar: AppBar(
        centerTitle: true,
        title: Text(currentProduct.name),
      ),
    );
  }
}
```

Bisa dilihat, data yang akan kita gunakan berasal dari provider. Keuntungannya adalah kita tidak perlu passing data melalui constructor. Selain itu, layar akan di-update setiap kali perubahan terjadi, sedangkan `productId` berasal dari arguments yang kita kirimkan pada saat menekan product.
